# design.md

word-list の設計ドキュメント。

---

## 設計思想

**入力から保存までが遠くなっていないこと。** これが最優先。
単語帳アプリとして「覚える機能」を増やす方向には進まない。気づいた瞬間に残せることが価値の中心にある。

### 判断基準

- その場ですぐ残せるか
- 単語とメモの対応が一目で分かるか
- 後から見返す負担が少ないか
- 機能を増やすほど、入力が遅くなっていないか

## アーキテクチャ概要

`index.html` / `script.js` / `styles.css` がエントリポイント。フレームワーク・ビルドステップなし。

```
word-list/
├── index.html          # アプリ本体
├── script.js           # ロジック全体
├── styles.css          # スタイル
├── sw.js               # Service Worker
├── manifest.webmanifest
└── docs/
    ├── product-principles.md
    └── design.md       # このファイル
```

デプロイは `docs/` フォルダを GitHub Pages のソースとして公開。`main` への push で即時反映。

## データ構造

### localStorage キー

| キー | 内容 |
|---|---|
| `word-list-items` | 単語リスト（JSON配列） |

### アイテムのスキーマ

```json
{
  "id": "uuid-v4",
  "word": "serendipity",
  "note": "思いがけない幸運に出会う能力",
  "tags": ["英語", "IELTS"],
  "checks": 2,
  "createdAt": "2026-01-15T10:30:00.000Z",
  "updatedAt": "2026-01-20T08:00:00.000Z"
}
```

- `id` — `crypto.randomUUID()` で生成
- `word` — 単語テキスト（最大80文字）。保存・検索時に `toLocaleLowerCase()` で正規化
- `note` — 意味・メモ（最大240文字）。任意
- `tags` — タグ名の配列。空配列可
- `checks` — 同じ単語を追加しようとした回数（初回追加で1）
- `createdAt` / `updatedAt` — ISO 8601文字列

### エクスポートのフォーマット

```json
{
  "version": 1,
  "exportedAt": "2026-01-20T08:00:00.000Z",
  "items": [...]
}
```

インポート時は `isValidItem` でバリデーションを行い、不正なデータを拒否する。ファイルサイズ上限は 2MB。

## 主要な動作

### 重複登録時のマージ

同じ単語（大文字小文字を無視）を追加すると、削除せずに `checks` をインクリメントする。
メモが入力されていれば既存のメモを上書きし、タグは既存タグに追加（重複排除）する。

### 編集時の重複マージ

編集フォームで別の単語と同名になった場合、そのまま更新せず既存エントリへマージする。
`checks` は合算され、メモ・タグも統合される。

## 今後の拡張方針・やらないこと

### やってよいこと

- 見返しやすさを上げるUI改善（ソート順変更など）
- テストモードの小改善（ランダム順など）
- アクセシビリティの改善

### やらないこと

- 復習スケジュール・記憶曲線の導入（他サービスの領域）
- 単語ごとの達成率・スコアリング
- クラウド同期（データはローカルのみ）
- 複雑なフォルダ階層やネスト構造
